## ITEM 1 — Region Dashboard Filters: Week + Sales Director

### What's missing
All 5 region tabs (N. CA, S. CA, TX, FL, FOR) are missing two filter dropdowns
that exist on every other filterable view in the portal since v0.5.14.

Current filter row (all 5 regions):
```
Month | Customer Type | Ship Type | Trailer Type | Site
```

Target filter row:
```
Month | Customer Type | Ship Type | Trailer Type | Site | Week | Sales Director
```

### Data fields
Both fields were already added to `extract_data.py` in v0.5.14:
- `week` — integer (ISO week number, e.g. 19, 20, 21)
- `sales_director` — string (e.g. "MIDWEST", "WEST COAST", "SOUTHEAST TX", "SOUTHEAST FL")

No extractor changes needed — just wire the dropdowns in the UI.

### Implementation
The region dashboards share a single template component. Add the two dropdowns
once and all 5 regions get them automatically.

```jsx
// Add to region filter state (alongside existing month/custType/shipType/trailerType/site)
const [weekFilter,      setWeekFilter]      = useState('All');
const [directorFilter,  setDirectorFilter]  = useState('All');

// Derive options from loaded data
const weekOptions     = ['All', ...unique(data.loads.map(r => r.week)).sort((a,b) => a - b)];
const directorOptions = ['All', ...unique(data.loads.map(r => r.sales_director)).sort()];

// Add to filter row JSX — after the existing Site dropdown
<FilterDropdown
  label="Week"
  value={weekFilter}
  options={weekOptions}
  onChange={setWeekFilter}
/>
<FilterDropdown
  label="Sales Director"
  value={directorFilter}
  options={directorOptions}
  onChange={setDirectorFilter}
/>
```

Add to the filter predicate wherever the other 5 filters are applied:
```javascript
&& (weekFilter     === 'All' || row.week           === weekFilter)
&& (directorFilter === 'All' || row.sales_director === directorFilter)
```

Both filters should drive every section on the region tab:
KPI cards, Site Breakdown, Monthly Trend, Freight Rings,
Trailer Utilization, and Opportunity Breakdown — same as Month and Site already do.
The 5-year efficiency metrics table is explicitly historical and ignores all filters
(same behavior as today — no change needed there).

## ITEM 2 — Build Health + Change Log Tabs

### Why these matter
These are **data trust and audit trail** features, not just metadata.
When the CEO or a distribution manager questions a number, these tabs answer:
- Is this data current? (source file name + ship date range)
- Did the pipeline validate cleanly? (verify gate result)
- When did the rates change? (change log with dates and detail)
- How many records are behind these numbers? (row counts, loads, drops)

### Data requirement — `extract_data.py` additions

Add a `build_health` object and `change_log` array to the freight JSON output.
The pipeline already has all this information; it just needs to be surfaced.

```python
import os, json
from datetime import datetime

# ── Build Health ──────────────────────────────────────────
build_health = {
    "generated_at":      datetime.now().strftime("%Y-%m-%d %H:%M"),
    "source_file":       SOURCE_FILE_NAME,        # str: filename of the .xlsb
    "source_size_mb":    round(SOURCE_FILE_MB, 1),
    "master_rows":       len(master_clean),
    "total_rows_before_exclusion": len(master_raw),
    "excluded_rows":     len(master_raw) - len(master_clean),
    "ship_date_min":     master_clean['ship_date'].min().strftime("%b %d, %Y"),
    "ship_date_max":     master_clean['ship_date'].max().strftime("%b %d, %Y"),
    "loads":             int(master_clean['tracking'].nunique()),
    "drops":             int(len(master_clean)),
    "pipeline_steps":    29,
    "verify_gate":       run_verify_gate(),        # returns "PASSED ✓" or "FAILED ✗ — <reason>"
    "bud_mile": {
        "BNL": 1.08, "BRA": 7.56, "FAL": 7.66,
        "GFL": 5.00, "MCR": 3.70, "PIR": 3.64,
        "STE": 7.66, "WIN": 7.56,
    },
    "kpis": {
        "total_cost":        round(float(master_clean['cost'].sum()), 2),
        "total_recovery":    round(float(master_clean['recovery'].sum()), 2),
        "net_recovery":      round(float(master_clean['net'].sum()), 2),
        "recovery_pct":      round(float(master_clean['recovery'].sum() /
                                        master_clean['cost'].sum()), 6),
        "total_miles":       round(float(master_clean['miles'].sum()), 3),
        "avg_cost_per_mile": round(float(master_clean['cost'].sum() /
                                        master_clean['miles'].sum()), 6),
        "avg_cost_per_load": round(float(master_clean['cost'].sum() /
                                        master_clean['tracking'].nunique()), 6),
        "threep_cost_per_mile": round(float(
            master_clean[master_clean['ship_type']=='3rd Party Freight']['cost'].sum() /
            master_clean[master_clean['ship_type']=='3rd Party Freight']['miles'].sum()), 6),
        "internal_cost_per_mile": round(float(
            master_clean[master_clean['ship_type']=='INTERNAL FREIGHT']['cost'].sum() /
            master_clean[master_clean['ship_type']=='INTERNAL FREIGHT']['miles'].sum()), 6),
    }
}

output['build_health'] = build_health

# ── Change Log ────────────────────────────────────────────
CHANGE_HISTORY_PATH = Path(__file__).parent / "change_history.json"

if CHANGE_HISTORY_PATH.exists():
    with open(CHANGE_HISTORY_PATH) as f:
        change_log = json.load(f)   # already newest-first array
else:
    change_log = []

output['change_log'] = change_log
```

### Build Health component

```jsx
// BuildHealth.jsx
export function BuildHealth({ data }) {
  const bh = data.build_health;
  if (!bh) return <Placeholder message="Build Health data not available" />;

  const gateColor = bh.verify_gate?.startsWith('PASSED') ? 'green' : 'red';

  return (
    <div className="build-health">

      {/* Verify gate — most important signal, show first */}
      <StatusBanner color={gateColor}>
        Verify Gate: {bh.verify_gate}
      </StatusBanner>

      {/* Two-column layout */}
      <div className="bh-grid">

        {/* Data Source & Extent */}
        <Section title="Data Source & Extent">
          <Row label="Source file"      value={bh.source_file} />
          <Row label="Source size"      value={`${bh.source_size_mb} MB`} />
          <Row label="Master rows"      value={`${fmt.n(bh.master_rows)} of
                                               ${fmt.n(bh.total_rows_before_exclusion)}
                                               (excluded ${fmt.n(bh.excluded_rows)} VMI)`} />
          <Row label="Ship dates"       value={`${bh.ship_date_min} → ${bh.ship_date_max}`} />
          <Row label="Loads"            value={fmt.n(bh.loads)} />
          <Row label="Drops"            value={fmt.n(bh.drops)} />
        </Section>

        {/* Build Status */}
        <Section title="Build Status">
          <Row label="Built"            value={bh.generated_at} />
          <Row label="Pipeline steps"   value={bh.pipeline_steps} />
          <Row label="Verify gate"      value={bh.verify_gate} color={gateColor} />
        </Section>

        {/* BUD $/Mile Rates */}
        <Section title="2026 Internal BUD $/Mile Rates">
          {Object.entries(bh.bud_mile).map(([site, rate]) => (
            <Row key={site} label={site} value={`$${rate.toFixed(2)}`} />
          ))}
          <Row label="FOR-region sites" value="actual (not budget)" muted />
        </Section>

        {/* Headline KPIs */}
        <Section title="YTD Headline KPIs">
          <Row label="Total Frt Cost"      value={fmt.dollar(bh.kpis.total_cost)} />
          <Row label="Total Frt Recovery"  value={fmt.dollar(bh.kpis.total_recovery)} />
          <Row label="Net Recovery"        value={fmt.dollar(bh.kpis.net_recovery)}
               color={bh.kpis.net_recovery < 0 ? 'red' : 'green'} />
          <Row label="Recovery % of Cost"  value={fmt.pct(bh.kpis.recovery_pct)} />
          <Row label="Total Miles"         value={fmt.n(bh.kpis.total_miles)} />
          <Row label="Avg $/Mile"          value={fmt.dollar(bh.kpis.avg_cost_per_mile)} />
          <Row label="Avg $/Load"          value={fmt.dollar(bh.kpis.avg_cost_per_load)} />
          <Row label="3P $/Mile YTD"       value={fmt.dollar(bh.kpis.threep_cost_per_mile)} />
          <Row label="Internal $/Mile YTD" value={fmt.dollar(bh.kpis.internal_cost_per_mile)} />
        </Section>

      </div>
    </div>
  );
}
```

### Change Log component

```jsx
// ChangeLog.jsx
const TYPE_COLORS = {
  FEATURE: 'navy',
  FIX:     'gold',
  DATA:    'green',
  RULE:    'purple',
};

export function ChangeLog({ data }) {
  const log = data.change_log || [];
  if (!log.length) return <Placeholder message="No change history available" />;

  return (
    <div className="change-log">
      <div className="cl-header">
        Change Log — {log.length} entries, newest first
      </div>
      {log.map((entry, i) => (
        <div key={i} className="cl-entry">
          <div className="cl-meta">
            <span className="cl-date">{entry.date}</span>
            <span className={`cl-type cl-type-${TYPE_COLORS[entry.type] || 'grey'}`}>
              {entry.type}
            </span>
          </div>
          <div className="cl-summary">{entry.summary}</div>
          {entry.detail && (
            <div className="cl-detail">{entry.detail}</div>
          )}
        </div>
      ))}
    </div>
  );
}
```

Type badge colors (match portal palette):
- `FEATURE` → navy (`#1F3A5F`)
- `FIX` → gold (`#C49B3F`)
- `DATA` → green (`#2F5233`)
- `RULE` → purple (`#5B4F8A`)
